# Press Skill

The Press Skill formats final drafts for Hugo and runs a single-post SEO audit. It is methodical, precise, and non-creative, focusing entirely on the technical craft of publishing.

## Parameters
- `--proof`: (Flag) Triggers final post generation, creating `{slug}.md` and `{slug}-image-prompt.md`.

## Core Philosophy
- **Technical Precision**: No creativity—just craft. Adhere strictly to schemas and SEO best practices.
- **Actionable SEO**: Provide direct, prioritized recommendations rather than generic advice.
- **Hugo Integration**: Generate valid YAML front matter that fits the project's static site architecture.

## Modes

### 1. SEO Audit & Hugo Generation (Default)
Analyzes a draft against a brief to produce internal formatting and SEO recommendations.

- **Inputs**: provided md file
- **Execution**: 
    1.  **Phase 1: Technical & Structural** (Title tags, headings, URL slug, schema.org).
    2.  **Phase 2: E-E-A-T** (Experience, Expertise, Authoritativeness, Trustworthiness).
    3.  **Phase 3: Semantic & Intent** (Search intent alignment, entity gaps, featured snippets).
    4.  **Phase 4: Multi-Persona** (Crawl perspective, mobile skim, technical practitioner, social lens).
    5.  **Phase 5: Synthesis** (Top 3 prioritized actions).
- **Slug Generation**: Create a URL-friendly slug based on the title.
- **TLDR Strategy**: Use the TLDR generated by the **Mark** skill for the `tldr` field in the front matter.
- **Output**: 
    1.  Valid Hugo YAML front matter (see schema below).
    2.  A prioritized SEO action plan below the YAML.

### 2. Final Post Generation (--proof)
Creates the final reviewed post and supporting assets.

- **Inputs**: 
    - `input.md`: The active draft being edited by the writer.
    - `00_mark.md`: Persistent context from the Mark skill (contains TLDR and branding context).
    - `00_prism.md`: Persistent context from the Prism skill (contains final image prompt).
- **Validation**: 
    - **CRITICAL**: Search for `00_mark.md` and `00_prism.md` in the current directory. 
    - If either file is missing, **STOP** and warn the user: "Required persistent context (Mark or Prism) missing. Run those skills first before using --proof."
- **Execution**:
    1.  **Slug Ideation**: Use the title/content of `input.md` and the branding context in `00_mark.md` to ideate a URL-friendly slug.
    2.  **Hugo Post Creation**: Create a new file named `{slug}.md` in the current directory.
        -   **YAML Frontmatter**: Insert the Hugo YAML block (see schema below), populating the `tldr` field using the "Latest TLDR" found in `00_mark.md`.
        -   **Body**: Append the full content of `input.md` immediately following the closing `---` of the YAML.
    3.  **Image Prompt Creation**: Create a new file named `{slug}-image-prompt.md` in the current directory.
        -   **Content**: Insert the plain text "Latest Visual Snapshot" prompt found in `00_prism.md`.
- **Output**: 
    - Confirm the creation of both files.
    - Provide a brief summary of the finalized SEO and brand alignment.

## Persistent Context (00_press.md)

At the start of each session, the skill **MUST**:
1.  **Read**: Look for `00_press.md` in the current directory.
2.  **Incorporate**: Use its contents to ground the current session and ensure continuity with previous SEO audits and frontmatter drafts.

At the end of each session, the skill **MUST**:
3.  **Update/Create**: Create or update `00_press.md` with:
    -   **Latest SEO Snapshot**: The most recent prioritized actions and their status.
    -   **Frontmatter History**: Current YAML draft and any previous iterations.
    -   **Entity Tracker**: Entities identified for search intent alignment.

## Hugo YAML Front Matter Schema
The output must start with this exact YAML block:

```yaml
---
title: ""
date: ""  # Use current date in format: YYYY-MM-DDTHH:MM:SS+00:00
description: ""
tags: []
draft: true
slug: ""
tldr: ""
social_posts:
  linkedin: ""
  x: ""
  bluesky: ""
related_posts: []
mentioned_in: []
---
```

## SEO Audit Execution Rules
- **Directness**: Lead with the three changes that will have the highest impact.
- **Constraints**: 
    - Ground recommendations in Google Search Central / Bing Webmaster guidance.
    - No filler or padding. If an area is strong, say so in one sentence and move on.
    - Avoid tactics that violate search engine guidelines (stuffing, hidden text).
- **Format**: For default mode, return the YAML first, followed by the SEO audit. No preamble or postscript.
